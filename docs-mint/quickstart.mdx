## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.11+**: The orchestrator requires Python 3.11 or higher
- **PostgreSQL**: A PostgreSQL database for storing orchestrator data
- **Git**: For cloning repositories (optional)

<Note>
We recommend using a virtual environment to isolate your orchestrator installation from other Python projects.
</Note>

## Step 1: Install the Package

Create a virtual environment and install the orchestrator-core package:

<CodeGroup>

```bash Terminal
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
pip install orchestrator-core
```

```bash Docker Alternative
# If you prefer Docker, you can use our example repository
git clone https://github.com/workfloworchestrator/example-orchestrator
cd example-orchestrator
docker-compose up -d
```

</CodeGroup>

## Step 2: Setup Database

Create a PostgreSQL database for the orchestrator:

<CodeGroup>

```bash Local PostgreSQL
createuser -sP nwa
createdb orchestrator-core -O nwa
# Enter password when prompted (remember it for step 4)
```

```bash Docker PostgreSQL
# Start a temporary PostgreSQL instance
docker run --rm --name temp-orch-db \
  -e POSTGRES_PASSWORD=rootpassword \
  -p 5432:5432 \
  pgvector/pgvector:pg17

# In another terminal, create user and database
docker exec -it temp-orch-db su - postgres -c \
  'createuser -sP nwa && createdb orchestrator-core -O nwa'
```

</CodeGroup>

<Warning>
Make sure to remember the password you set for the `nwa` user - you'll need it in the next steps.
</Warning>

## Step 3: Create Application

Create a `main.py` file in your project directory:

```python main.py
from orchestrator import OrchestratorCore
from orchestrator.cli.main import app as core_cli
from orchestrator.settings import app_settings

app = OrchestratorCore(base_settings=app_settings)

if __name__ == "__main__":
    core_cli()
```

<Tip>
This minimal setup creates a fully functional orchestrator application with all core features enabled.
</Tip>

## Step 4: Initialize Database

Set up your database connection and run migrations:

```bash
export DATABASE_URI=postgresql://nwa:YOUR_PASSWORD@localhost:5432/orchestrator-core

python main.py db init
python main.py db upgrade heads
```

<Note>
Replace `YOUR_PASSWORD` with the password you set for the `nwa` user in Step 2.
</Note>

## Step 5: Start the Application

Launch your orchestrator application:

```bash
export DATABASE_URI=postgresql://nwa:YOUR_PASSWORD@localhost:5432/orchestrator-core
export OAUTH2_ACTIVE=False  # Disable OAuth for development

uvicorn --reload --host 127.0.0.1 --port 8080 main:app
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database Connection Issues">
    - Ensure PostgreSQL is running and accessible
    - Verify the DATABASE_URI format: `postgresql://user:password@host:port/database`
    - Check that the `nwa` user has the correct permissions
  </Accordion>

  <Accordion title="Port Already in Use">
    If port 8080 is already in use, change it in the uvicorn command:
    ```bash
    uvicorn --reload --host 127.0.0.1 --port 8081 main:app
    ```
  </Accordion>

  <Accordion title="Python Version Issues">
    Ensure you're using Python 3.11 or higher:
    ```bash
    python --version
    ```
    If needed, install a newer Python version or use pyenv to manage multiple versions.
  </Accordion>
</AccordionGroup>
