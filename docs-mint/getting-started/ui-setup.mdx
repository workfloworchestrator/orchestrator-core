## Overview

The Workflow Orchestrator UI provides a modern web interface for managing subscriptions, executing workflows, and monitoring your services. This guide will help you set up the UI alongside your orchestrator backend.

<Note>
The UI is a separate React application that communicates with the orchestrator backend via REST and GraphQL APIs.
</Note>

## Prerequisites

Before setting up the UI, ensure you have:

- **Orchestrator Backend**: Running on `http://localhost:8080`
- **Node.js 18+**: For running the React application
- **npm or yarn**: Package manager for JavaScript dependencies

## Step 1: Install the UI

<Tabs>
  <Tab title="Using npm">

## Step 2: Configure the UI

Create or update your `.env` file:

```bash .env
# Backend API configuration
REACT_APP_ORCHESTRATOR_API_URL=http://localhost:8080
REACT_APP_GRAPHQL_ENDPOINT=http://localhost:8080/api/graphql

# Authentication (if OAuth is enabled)
REACT_APP_OAUTH_ENABLED=false
REACT_APP_OAUTH_CLIENT_ID=orchestrator-ui
REACT_APP_OAUTH_AUTHORIZATION_URL=https://auth.example.com/oauth2/authorize
REACT_APP_OAUTH_REDIRECT_URI=http://localhost:3000/auth/callback

# UI Configuration
REACT_APP_ORGANIZATION_NAME=Your Organization
REACT_APP_THEME_PRIMARY_COLOR=#0D9373
REACT_APP_LOGO_URL=/logo.png

# Feature flags
REACT_APP_ENABLE_GRAPHQL_PLAYGROUND=true
REACT_APP_ENABLE_WORKFLOW_BUILDER=true
REACT_APP_ENABLE_SUBSCRIPTION_SEARCH=true
```

## Step 3: Start the UI

```bash
# Development mode with hot reload
npm start

# Production build
npm run build
npm run serve
```

The UI will be available at `http://localhost:3000`.

## Step 4: Configure Backend for UI

Update your orchestrator backend to support the UI:

```python main.py
"""Updated main.py with UI support."""

from orchestrator import OrchestratorCore
from orchestrator.cli.main import app as core_cli
from orchestrator.settings import app_settings

# Configure CORS for UI
app_settings.CORS_ORIGINS = [
    "http://localhost:3000",  # Development UI
    "https://orchestrator.example.com"  # Production UI
]
app_settings.CORS_ALLOW_CREDENTIALS = True

# Enable GraphQL playground in development
app_settings.GRAPHQL_PLAYGROUND_ENABLED = True

# Initialize orchestrator with UI support
app = OrchestratorCore(
    base_settings=app_settings,
    enable_graphql=True,  # Required for UI
    enable_websockets=True  # For real-time updates
)

if __name__ == "__main__":
    core_cli()
```

## Step 5: Verify Integration

### Check API Connectivity

1. **Open the UI**: Navigate to `http://localhost:3000`
2. **Check Health**: The UI should show a green status indicator
3. **View Subscriptions**: Navigate to the subscriptions page
4. **Test GraphQL**: Open the GraphQL playground at `/graphql`

### Test Workflow Execution

1. **Navigate to Workflows**: Click on "Workflows" in the sidebar
2. **Select Workflow**: Choose "Create Internet Service"
3. **Fill Form**: Complete the service creation form
4. **Execute**: Click "Start Workflow" and monitor progress
5. **View Result**: Check the created subscription in the subscriptions list

## Production Deployment

### Build for Production

```bash
# Create production build
npm run build

# The build folder contains static files ready for deployment
ls -la build/
```

### Nginx Configuration

```nginx nginx.conf
server {
    listen 80;
    server_name orchestrator-ui.example.com;

    root /var/www/orchestrator-ui/build;
    index index.html;

    # Handle React Router
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API proxy to backend
    location /api/ {
        proxy_pass http://orchestrator-backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket support for real-time updates
    location /ws {
        proxy_pass http://orchestrator-backend:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # Static assets caching
    location /static/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### Docker Deployment

```dockerfile Dockerfile.ui
FROM node:18-alpine as builder

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/build /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="UI Cannot Connect to Backend">
    **Symptoms**: UI shows "Backend Unavailable" message

    **Solutions**:
    - Verify backend is running on correct port
    - Check CORS configuration in backend
    - Ensure `REACT_APP_ORCHESTRATOR_API_URL` is correct
    - Check browser console for network errors
  </Accordion>

  <Accordion title="Authentication Issues">
    **Symptoms**: Login fails or redirects incorrectly

    **Solutions**:
    - Verify OAuth configuration matches backend
    - Check redirect URI is registered with OAuth provider
    - Ensure `REACT_APP_OAUTH_REDIRECT_URI` matches actual URL
    - Check browser cookies and local storage
  </Accordion>

  <Accordion title="GraphQL Errors">
    **Symptoms**: Data not loading, GraphQL errors in console

    **Solutions**:
    - Verify GraphQL endpoint is accessible
    - Check GraphQL playground at `/api/graphql`
    - Ensure backend has GraphQL enabled
    - Check authentication headers for GraphQL requests
  </Accordion>

  <Accordion title="Build Errors">
    **Symptoms**: `npm run build` fails

    **Solutions**:
    - Clear node_modules and reinstall: `rm -rf node_modules && npm install`
    - Check Node.js version compatibility
    - Verify all environment variables are set
    - Check for TypeScript errors: `npm run type-check`
  </Accordion>
</AccordionGroup>
